Therapeutic Candidate 1:
Candidate: Graph-Constrained Semantic Routing (GCSR) for Multi-File Context
Hypothesis: Integrating explicit Abstract Syntax Tree (AST) dependency constraints into the vector retrieval process significantly reduces "contextual noise" (irrelevant code fragments) in multi-file software engineering tasks, leading to higher F1-scores in code completion benchmarks compared to pure semantic similarity retrieval.
Reasoning: Current RAG systems for software engineering typically rely on embedding-based similarity, which often retrieves semantically similar code that is functionally disconnected from the target module. By constructing an "Import-Usage" graph and using it as a hard constraint or re-ranking filter (a "graph-gated" approach), we can prune the search space to only those files reachable within the dependency chain. The novelty lies in the hybrid architectural approach, shifting from "what code looks like" to "what code interacts with." Evaluation can be conducted using the HumanEval-X multi-file subset, measuring metrics like "Dependency Coverage Ratio" and "Syntactic Correctness" using tree-sitter to validate the output code.

Therapeutic Candidate 2:
Candidate: Mutation-Based Adversarial Benchmarking (MBAB) for Dependency Resolution
Hypothesis: RAG systems for code generation exhibit higher hallucination rates when dependency paths are perturbed via subtle "dependency-injection" mutations (e.g., renaming a variable in a linked library file) than when purely semantic, non-functional code fragments are injected.
Reasoning: The lack of ground truth datasets that specifically target dependency-awareness is a major gap. This proposal introduces a framework to systematically mutate software repositories to create "adversarial retrieval" scenarios. By intentionally breaking or modifying the dependency links in a codebase and observing if the LLM retrieves the stale or the mutated dependency, we can quantify "Retrieval Fidelity." This method provides a causal way to measure how robust an RAG system is against "dependency drift" in large-scale repositories. Metrics would include "Dependency Resolution Success Rate" (DRSR) rather than just BLEU or CodeBLEU.

Therapeutic Candidate 3:
Candidate: Causal Ablation Framework for Retrieval Components (CAFRC)
Hypothesis: The performance variance observed in RAG-based code review is driven more by the "depth" of the dependency graph traversal than by the raw dimensionality of the vector embeddings used for retrieval.
Reasoning: To isolate the impact of different retrieval components, we need a causal framework that treats the retrieval architecture (e.g., Graph-RAG vs. Vector-RAG) as a treatment variable in a controlled experiment. By utilizing propensity score matching on code repository complexity (Cyclomatic complexity, graph connectivity), we can isolate whether the LLM's improvement comes from the quality of retrieval or the prompt engineering around it. This addresses the challenge of confounding variables in RAG evaluation. The feasibility is high if we leverage existing codebases like `RefactoringMiner` or `CodeSearchNet` and apply Do-calculus to estimate the causal effect of retrieval modules on downstream code generation accuracy.

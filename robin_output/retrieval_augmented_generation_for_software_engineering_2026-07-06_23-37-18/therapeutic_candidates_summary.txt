Therapeutic Candidate 1:
Candidate: Topological Context Pruning (TCP) for Multi-Repo RAG
Hypothesis: In multi-repository settings, retrieving code context based on "topological distance" (n-hop traversal in a global dependency graph) significantly reduces LLM context noise and improves API integration accuracy compared to traditional vector-similarity-based retrieval.
Reasoning: Current RAG systems often suffer from the "lost in the middle" phenomenon when fetching code from large-scale, multi-repo environments because vector embeddings capture linguistic similarity rather than the structural necessity of a code segment. By implementing an index based on the call graph/dependency graph, we can perform targeted traversal (e.g., retrieving only the direct dependents or definitions of a targeted API). This approach leverages the "world-side compilation" [4] theory, treating the repo as a graph to be traversed rather than a document set to be scanned. Feasibility is high given existing AST-to-graph parser tools; novelty lies in moving from "flat" retrieval to "traversal-based" context window population.

Therapeutic Candidate 2:
Candidate: Semantic Entropy-Based Hallucination Detection (SEHD) for API Usage
Hypothesis: The occurrence of hallucinated API calls during RAG-assisted code generation can be significantly mitigated by measuring the "semantic entropy" of the generated code chunks across multiple stochastic decoding paths, specifically flagging code sequences that deviate from the inferred dependency schema of the target repository.
Reasoning: Standard code-accuracy benchmarks (e.g., Pass@k) fail to detect subtle hallucinations where an LLM invokes an API that "looks" correct but violates the dependency constraints of the specific multi-repository environment. By applying semantic entropy [6], we can quantify model uncertainty during generation. If the model generates a non-existent API call, the entropy across sampled paths will spike, providing a real-time signal for an agentic "tester" role [5] to trigger re-retrieval. This provides a robust, task-agnostic evaluation metric that bridges the gap between general reasoning and domain-specific correctness.

Therapeutic Candidate 3:
Candidate: Topology-Semantic Ablation (TSA) Framework for Modular RAG
Hypothesis: The relative contribution of graph-based topological context versus vector-based semantic context can be quantitatively isolated in API integration tasks by evaluating performance in a modular RAG pipeline where the graph-retrieval and semantic-retrieval modules are toggled independently.
Reasoning: Despite the growing popularity of GraphRAG, it remains unclear whether the performance gains come from the graph structure or the improved document chunking. This proposal advocates for a rigorous, LEGO-like modular architecture [4] to perform an ablation study. By isolating these modalities, we can determine the "tipping point" where structural dependency information becomes more critical than semantic similarity (e.g., in legacy refactoring tasks versus greenfield feature implementation). This addresses the critical research need for moving beyond qualitative evaluations of RAG architectures toward formal validation of the information channels that actually drive software engineering productivity.
